# Story 1.4: Develop Contact Page

## Status
Done

## Story
**As a** Visitor to the Paradise Dental website,
**I want** to view contact information, location details, and an embedded map,
**so that** I can easily find the clinic's address, phone number, email, and operating hours.

## Acceptance Criteria
1. Contact page displays clinic name and page heading with brand-compliant styling
2. Contact information section shows phone number, email, and physical address
3. Operating hours are displayed clearly (weekday and weekend schedules)
4. Embedded Google Maps iframe showing clinic location
5. Page uses pure white background (#FFFFFF) throughout
6. All text uses Prompt font family exclusively
7. Brand colors applied correctly: #214491 for headings, #2D7C9C for icons, #CEE0F3 for section backgrounds
8. Contact information cards use proper card styling (white background, subtle borders, soft shadows)
9. Icons for phone, email, and location use Lucide React (line style) in #2D7C9C
10. Page is fully responsive (mobile-first design) - contact cards stack vertically on mobile
11. Navigation component (Header) is integrated and functional
12. Footer component is integrated
13. All interactive elements (email/phone links) are clickable with proper hover states
14. Map is responsive and adjusts height on mobile devices

## Dev Notes

### UX Design Validation
*(Sally, UX Expert)*

The contact page is a critical point of conversion. Our goal is to make it as frictionless and reassuring as possible for a user to get in touch.

**1. Information Hierarchy & Scannability:**
*   **Above the Fold:** The most critical contact details (Phone, Email, Address) should be immediately visible without scrolling. The 3-card layout is perfect for this.
*   **Iconography:** The choice of `lucide-react` is excellent for its clean, light style. Ensure the icons are large enough to be immediately recognizable within their circular containers. A size of `h-8 w-8` for the icon within a `h-16 w-16` container is a good proportion.
*   **Clear Labeling:** The "Phone", "Email", "Location" labels are clear and direct. Keep them.

**2. Interaction & Affordance:**
*   **Clickable Cues:** The `tel:` and `mailto:` links are essential. Let's make the entire card clickable, not just the text link, to provide a larger touch target, especially on mobile. We can wrap each card's content in an `<a>` tag.
*   **Hover/Focus States:** The `hover:-translate-y-1` effect should apply to all cards (Contact Info, Operating Hours) for consistency. When a card is focused (via keyboard tabbing), it should have the same visual prominence (e.g., the shadow and transform effect) as the hover state.
*   **Map Interaction:** The embedded map should feel like part of the page. The `rounded-xl` style on the iframe is a nice touch that matches the card aesthetic. On hover, we could add a slightly stronger border to the map container to indicate it's an interactive element.

**3. Building Trust & Confidence:**
*   **Operating Hours:** Placing the operating hours in a separate, clearly defined card is smart. It answers a common user question ("Are they open now?") without cluttering the primary contact methods.
*   **Physical Address:** Showing the full physical address builds significant trust. It signals that this is a legitimate, established clinic.
*   **CTA Placement:** The final "Book Your Appointment" CTA at the bottom of the page is a great way to re-engage the user after they've reviewed the contact information. It provides a clear next step.

**4. Mobile Experience:**
*   **Card Stacking:** On mobile, the cards should stack vertically with a clear margin between them (`mb-6` or `mb-8`).
*   **Map Height:** The reduced map height (`h-[300px]`) on mobile is a crucial optimization to prevent the map from dominating the viewport.
*   **Click-to-Call:** The `tel:` link is most important on mobile. Ensure it's easily tappable.

By focusing on these interaction details and trust signals, the contact page will effectively convert user interest into actual contact.

### Accessibility Considerations
[Source: General web standards - WCAG AA compliance]

**Required for Contact Page:**
- Proper heading hierarchy (H1 → H2)
- Alt text for any decorative images (if used)
- Clickable phone/email links with clear focus states
- Icon labels for screen readers (aria-label or sr-only text)

- Touch targets at least 44px × 44px on mobile
- Keyboard navigable (Tab, Enter for links/buttons)
